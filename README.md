# Human Activity Recognition
Human Activity Recognition with Optimized Machine Learning Models

A machine learning project for PV056 — Machine Learning and Data Mining at Masaryk University.

We classify 6 human activities from smartphone sensor data using Logistic Regression, Random Forest, XGBoost, and a Bidirectional LSTM — with full hyperparameter optimization via Optuna.

## Results

| Model                     | Baseline F1-macro | HPO F1-macro |
|-------                    |:-----------------:|:------------:|
| **Logistic Regression**   | **0.9553**        | 0.9423       |
| Naive LSTM                | 0.9482            |    —         |
| XGBoost                   | 0.9429            | 0.9482       |
| Random Forest             | 0.9249            | 0.9216       |
| Improved LSTM (BiLSTM)    |          —        | 0.9501       |

Key finding: Logistic Regression achieved the highest overall F1-macro (0.9553), outperforming all models including the improved LSTM. This indicates the 561 handcrafted features are largely linearly separable — no complex temporal modeling is required when the feature engineering is already strong.

Notebook Walkthrough

1. Dataset Loading & Preprocessing


Loads train.csv (7,352 samples) and test.csv (2,947 samples)
Data quality checks: zero missing values, zero duplicates, no subject overlap between splits, zero zero-variance features
Label encoding: 6 activity strings → integers 0–5
Z-normalization using StandardScaler — fitted on training data only to avoid data leakage
Saves processed arrays to processed/har_train_test.npz for downstream sections


2. Exploratory Data Analysis


Class distribution bar charts — mild imbalance (ratio ~1.43)
Static vs dynamic activity comparison
Boxplots of representative features — standard deviation features separate activities better than mean features
Correlation heatmap — reveals high redundancy among the 561 features
PCA and t-SNE projections — dynamic activities form well-separated clusters; Sitting and Standing overlap significantly, identifying them as the hardest classification pair
Top 10 features by variance, subject distribution across 30 participants


3. Baseline Models

Four models trained with default settings to establish benchmarks:


Logistic Regression — linear model, 0.9553 F1-macro
Random Forest — 100 trees, 0.9249 F1-macro
XGBoost — gradient boosting, 0.9429 F1-macro
Naive LSTM — single LSTM layer, input shape (batch, 1, 561), 0.9482 F1-macro


The Naive LSTM is called "naive" because it receives each sample as a single timestep with no real sequence — essentially behaving like a dense neural network.

4. Hyperparameter Optimization

All models tuned using Optuna with a Tree-structured Parzen Estimator (TPE) sampler and 3-fold cross-validation, optimizing for F1-macro:

| Model               | Trials | Key parameters tuned                                                                                |
|-------              |:------:|---------------------                                                                                |
| Logistic Regression | 20     | C, solver, max_iter                                                                                 |
| Random Forest       | 5      | n_estimators, max_depth, min_samples_split, min_samples_leaf                                        |
| XGBoost             | 40     | learning_rate, max_depth, subsample, colsample_bytree, gamma, reg_alpha, reg_lambda                 |
| Improved LSTM       | 15     | n_layers (1–2), units (64/128/256), dropout (0.2–0.5), learning_rate, batch_size, bidirectionality  |

Improved LSTM architecture:


Input shape: (batch, 10, 561) — 10 consecutive windows as a real sequence (sliding window with step=1)
1 or 2 Bidirectional LSTM layers + Dropout after each
Dense output layer with 6 units and softmax activation
Adam optimizer, sparse categorical cross-entropy loss
Early stopping (patience=8) — stopped at ~epoch 19


5. Results & Error Analysis


Full model comparison: baseline vs HPO-optimized versions
Confusion matrix analysis — main errors concentrated in WALKING_UP vs WALKING_DOWN (88 total) and STANDING vs LAYING (21 errors)
Key insight: intensive HPO on high-dimensional data caused slight overfitting in RF, explaining why its HPO version underperformed the baseline
